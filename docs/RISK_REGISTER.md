# Risk Register

Status: active pre-development risk register.

Severity uses `critical`, `high`, `medium`, and `low`. `GA blocker` means the
risk must be closed before GA or accepted through the restricted residual-risk
process in `docs/DEVELOPMENT_GOVERNANCE.md`. Risks that can cause credential
exposure, tenant isolation failure, user data loss, irrecoverable operation
ambiguity, or caller-visible contract break are non-waivable for GA.
