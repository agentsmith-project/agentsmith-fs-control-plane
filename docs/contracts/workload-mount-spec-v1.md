# Contract: Workload Mount Spec V1

Status: draft

Workload mount spec is generated by AFSCP and consumed by an external orchestrator.

## Fields

- `namespace_id`
- `repo_id`
- `volume_id`
- `payload_subdir`
- `mount_path`
- `read_only`
- `secret_ref`

## Rules

- `payload_subdir` is generated by AFSCP.
- In P0, `payload_subdir` is the JVS `main` workspace real folder, not a child named `workspace`.
- workload containers receive no JuiceFS root credentials.
- workload containers must not be able to read or write `.jvs`.
- the external orchestrator does not decide product authorization.
