# Contract: Sandbox Mount Spec V2

Status: draft

Sandbox mount spec v2 is generated by AFSCP and consumed by sandbox-manager.

## Fields

- `tenant_workspace_id`
- `storage_repo_id`
- `filesystem_id`
- `storage_pool_id`
- `payload_subdir`
- `mount_path`
- `read_only`
- `secret_ref`

## Rules

- `payload_subdir` is generated by AFSCP.
- In P0, `payload_subdir` is the JVS `main` workspace real folder, not a child named `workspace`.
- workload containers receive no JuiceFS root credentials.
- workload containers must not be able to read or write `.jvs`.
- sandbox-manager does not decide user authorization.
- v1 remains for legacy libraries.
